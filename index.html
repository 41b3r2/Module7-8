What is Malware?
Malicious Software (Malware) — a wide range of different software with malicious intent. Includes many types targeting systems, data, and users.
Types of Malware
Buffer Overflow — programming error; writes beyond a fixed buffer into adjacent memory, allowing malicious code injection
Viruses — set of malicious code that infects a host; executes when an application runs; replicates and delivers its payload upon a trigger
Worms — travels the network autonomously without a host app or user interaction; consumes bandwidth; can slow networks to a crawl
Trojan Horse — disguises itself as something desirable (e.g., free antivirus, screen saver) but installs malware when executed
Logic Bomb — code embedded in an app that triggers on a specific event (date, program launch, or any programmer-defined condition)
Spyware — installs without consent; silently collects keystrokes, screenshots, credentials, PII, and web form data
Ransomware — encrypts victim files and demands payment (often crypto) to restore access
Adware — shows unwanted ads or collects marketing data; typically hidden until pop-ups appear
Rootkit — gains root/admin access via privilege escalation; modifies core system files to remain invisible and persist without detection
Backdoor — creates an alternative entry point that bypasses security controls; hidden access for attacker
Zero Day Attacks — exploits unknown software flaws with no patch; occurs on the day the weakness is discovered before a fix exists
Types of Viruses
Boot SectorInfects the boot sector of the hard drive; executes before the OS loads
ProgramEmbedded directly into an executable program file
ScriptWeb-based scripting language virus; often delivered through websites
MacroExploits Microsoft Office macros; spreads via documents
PolymorphicMutating virus; changes its own code to evade antivirus detection
📧 Email is the most popular method used to spread viruses. A virus requires a host application and user execution — unlike a worm.
Buffer Overflow
DefinitionA programming error where a program writes more data to a fixed-length buffer than it can hold; overflow spills into adjacent memory locations
ImpactAdjacent memory can be exploited with malicious code to crash the app/system or inject malware
ProtectionsData input validation, Windows run-time protections, secure development practices

Protecting Against Malware
Antivirus SoftwareSpecifically designed to detect viruses, trojans, worms, rootkits, spyware, and adware
Anti-spywareEmerged as a separate application specifically targeting spyware (separate from antivirus)
Bug BountyPrograms offered by software developers; individuals receive recognition/compensation for reporting bugs and vulnerabilities (related to zero-day mitigation)
Social Engineering
Social Engineering bypasses technology protections by using psychological tactics to encourage a person to perform a specific action or give up crucial information.
Mitigating Social Engineering
User Training & Awareness
Be suspicious & cautious of unsolicited requests
Verify someone's identity before sharing info
Don't rely on email alone as verification
Spam & Spoofed Email
Spam EmailUnsolicited emails — commonly advertising but sometimes phishing/scamming attempts; can carry malware
Email SpoofingForging the email header so the message appears to come from a legitimate source; used in phishing, pharming, and spam campaigns
PhishingSending deceptive email to trick users into revealing personal information (e.g., bank credentials) or clicking malicious links
Common Attack Methods
DoS AttackMalicious user floods a server with requests, overwhelming its resources so it can't respond to legitimate requests
DDoS AttackDoS launched from a large number of malicious machines simultaneously; harder to block
Backdoor AttackCreating an alternative way into a system that bypasses its security controls
Replay AttackAttacker captures a message from a device to a server, then resends the original unmodified message to create a "trusted" relationship
Weak EncryptionOccurs when enough traffic is captured to break the key; e.g., WEP encryption is vulnerable
Software Vuln AttackExploitation of known software/application vulnerabilities for malicious purposes
Remote Code ExecutionUsed against improperly coded web apps; allows attackers to run system-level code remotely
SQL InjectionMalicious SQL inserted via web input forms; can retrieve, delete, or drop database tables
XSS AttackMalicious HTML/JS injected into a website's code; executes when a victim visits the page; steals cookies, session data
Types of DoS/DDoS
Buffer OverflowsSending the server more data than expected
SYN AttackExploits the TCP three-way handshake
Ping of DeathExploits the ICMP "ping" protocol with oversized packets

Security Testing vs Assessment vs Audit
Security TestingVerifies that security controls are functioning properly; should take place on a regular schedule; focused on specific controls
Security AssessmentComprehensive review of security of systems; goes beyond testing; includes reviews of risk and the threat environment; assessment report is often a requirement
Security AuditVery similar to assessments but performed by independent auditors; can be internal, external, or third-party
🔑 Key distinction: Testing = verify controls. Assessment = comprehensive + risk review. Audit = assessment done by independent auditors.

Criticality of the systems being protected
Sensitivity of data on systems
Likelihood of technical failure
Likelihood of a misconfiguration
Risk the system will come under attack
Rate of change of the control configuration
Other changes in the technical environment
Difficulty and time required to perform the test
Impact of test on normal business operations
NIST SP 800-53A — Assessment Framework
NIST Special Publication 800-53A: "Assessing Security and Privacy Controls in Federal Information Systems and Organizations: Building Effective Assessment Plans"
4 Components of NIST Assessments
SpecificationsThe documents associated with the system being audited
MechanismsThe controls used within an information system to meet the specifications
ActivitiesThe actions carried out by people within the information system
IndividualsThe people who implement specifications, mechanisms, and activities
Types of Security Audits
Internal AuditPerformed by internal staff and typically for internal audiences; the reporting line is independent of the functions being evaluated
External AuditPerformed by an outside auditing firm
Third-Party AuditConducted by another organization; e.g., a regulatory body auditing a firm under their regulation
Audit Locations
On-PremisesAudit performed physically at the site
CloudAudit performed on cloud-hosted systems
HybridMix of on-premises and cloud audit
📋 COBIT (Control Objectives for Information and Related Technologies) is the most commonly used auditing standard framework.
Vulnerability Assessment
PurposeDiscover system weaknesses; tests are commonly automated and help build a remediation roadmap for discovered vulnerabilities
SCAPSecurity Content Automation Protocol — created by NIST; provides standards around the area of vulnerabilities
CVECommon Vulnerabilities and Exposures — standardized identifiers for publicly known vulnerabilities
CVSSCommon Vulnerability Scoring System — scores and ranks the severity of vulnerabilities
Types of Vulnerability Scans
Network Discovery Scans — identify devices and open ports on the network
Network Vulnerability Scans — detect security weaknesses in network infrastructure
Web App Vulnerability Scans — identify flaws in web applications (e.g., XSS, SQLi)
Database Vulnerability Scans — find weaknesses in database configurations and access
🎯 The main output of a vulnerability assessment = a Remediation Roadmap provided to stakeholders.

Key Performance & Risk Indicators (KPIs / KRIs)
Number of Open Vulnerabilities — how many unpatched or unresolved vulnerabilities exist
Time to Resolve — how quickly identified issues are remediated
Rate of Occurrence — how frequently the same type of security event occurs
Number of Compromises — how many accounts/systems were actually compromised
Number of Software Flaws — code defects found during security testing
Audit Findings — documented issues uncovered during an audit review
Analyzing Test Output — Report Components
The primary goal of test data analysis = produce a Remediation Roadmap for various stakeholders.
Executive Summary — high-level overview for leadership and non-technical stakeholders
Scope and Objectives — what was tested and why
Methodology — how the testing was conducted
Findings — discovered vulnerabilities and security issues
Risk Analysis — impact and likelihood assessment of findings
Recommendations — specific steps to fix identified issues
Timelines and Resources — remediation schedule and resource requirements
Exception Handling & Ethical Disclosure
ExceptionWhen a policy, procedure, or control is temporarily bypassed; must be done for legitimate reasons; always documented; exceptions always carry increased risk
Ethical DisclosureSome incidents must be ethically disclosed; many organizations lack a disclosure policy; DHS created a Vulnerability Disclosure Policy template to help
⚠️ Exceptions = temporary bypass = increased risk. Documentation is always key.
